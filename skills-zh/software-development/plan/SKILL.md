# Plan Mode（计划模式）

当用户想要计划而不是执行时，使用本技能。

## 核心行为

本轮只做规划。

- 不实现代码
- 不编辑项目文件（计划 markdown 文件除外）
- 不运行有副作用的终端命令、不提交、不推送、不执行外部操作
- 可以用只读命令/工具检查仓库或其他上下文
- 最终交付物是一份保存到当前工作区 `.hermes/plans/` 目录下的 markdown 计划文件

## 输出要求

写一份具体可执行的 markdown 计划。

相关内容包括：
- 目标（Goal）
- 当前上下文 / 假设
- 建议方案
- 逐步计划
- 可能改动的文件
- 测试 / 验证方式
- 风险、权衡和开放问题

如果是代码相关任务，要包含精确的文件路径、可能的测试目标，以及验证步骤。

## 保存位置

用 `write_file` 保存到：
- `.hermes/plans/YYYY-MM-DD_HHMMSS-<slug>.md`

路径相对于当前工作目录 / 后端工作区。Hermes 的文件工具是后端感知的，所以在这种相对路径下，计划会跟随工作区（本地、docker、ssh、modal、daytona 后端均适用）。

如果运行时提供了特定的目标路径，使用那个精确路径。
否则，在 `.hermes/plans/` 下自行创建带时间戳的文件名。

## 交互风格

- 如果请求足够清晰，直接写计划
- 如果没有明确指令，从当前对话上下文推断任务
- 如果确实描述不清，简要提问澄清，而不是猜测
- 保存计划后，简短回复计划内容和保存路径
